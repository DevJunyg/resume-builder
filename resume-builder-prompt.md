# AI 이력서 빌더 프로젝트 지시서

> 이 문서는 Claude Code 에이전트가 "AI 이력서 빌더" 프로젝트를 자율적으로 개발하기 위한 전체 사양서입니다. 각 섹션을 순서대로 참고하여 작업을 수행하세요.

---

## 프로젝트 개요

### 제품 컨셉
**대화하면서 완성되는 AI 이력서 빌더** — 사용자가 채팅으로 경력과 경험을 입력하면 AI가 STAR 기법으로 재구성하여 실시간 프리뷰로 이력서를 완성해주는 웹 도구. 채용공고(JD)를 붙여넣으면 해당 공고에 맞춰 이력서를 최적화하는 기능을 제공합니다.

### 차별점
- 기존 정적 템플릿 기반 이력서 도구와 달리, **대화형 인터페이스**로 경험을 끌어냄
- **JD 기반 맞춤 최적화** — 지원 회사마다 이력서를 자동으로 튜닝
- **스트리밍 기반 실시간 프리뷰** — 입력과 동시에 이력서가 완성되어가는 경험
- AI 응답을 **Diff 하이라이트**로 시각화하여 어디가 바뀌는지 즉시 파악 가능

### 프로젝트 목적
이 제품은 포트폴리오로서 다음을 증명하는 것이 최우선 목표입니다:
1. AI API를 단순 래핑하는 수준이 아닌, **AI를 중심으로 UX를 설계**할 수 있다
2. **스트리밍, Tool Use, 프롬프트 엔지니어링**을 실전 수준으로 다룰 수 있다
3. **FE 개발자만 구현할 수 있는 디테일**을 챙길 수 있다

## 핵심 기능

### MVP 필수 기능
1. **3단 레이아웃 UI**
   - 좌측: 채팅 인터페이스
   - 중앙: 이력서 실시간 프리뷰
   - 우측: 도구 패널 (JD 입력, 톤 조절, 내보내기)

2. **대화형 이력서 작성 플로우**
   - AI가 질문을 던지며 사용자의 경력/경험을 단계적으로 수집
   - 사용자의 답변을 STAR 기법(Situation-Task-Action-Result)으로 자동 재구성
   - Tool Use를 활용하여 구조화된 JSON 형태로 이력서 데이터 생성

3. **실시간 스트리밍 프리뷰**
   - Claude API의 스트리밍 응답을 이력서 프리뷰에 즉시 반영
   - 스트리밍 중에도 사용자가 다른 섹션 편집 가능

4. **JD 기반 최적화**
   - 우측 패널에 채용공고 붙여넣기
   - AI가 "이 공고에는 이 경험을 강조하는 것이 좋습니다" 제안
   - 강조된 섹션은 시각적으로 하이라이트

5. **섹션별 편집 기능**
   - 각 섹션별로 인라인 편집 가능
   - 섹션 드래그 앤 드롭으로 순서 변경

6. **PDF 내보내기**
   - 인쇄 최적화된 CSS 적용
   - `@react-pdf/renderer` 또는 `react-pdf` 활용

### FE 개발자 강점이 드러나는 디테일 (필수 구현)
- **스트리밍 Diff 하이라이트** — AI 응답으로 변경되는 부분을 애니메이션으로 표시
- **섹션별 Undo/Redo** — 섹션 단위로 변경 이력 관리
- **"이 문장 더 강하게" 우클릭 컨텍스트 메뉴** — 선택한 텍스트를 AI로 재작성
- **스켈레톤 + "AI가 생각 중..." 마이크로카피** — 로딩 상태 UX
- **AbortController 기반 스트리밍 취소** — 뒤로가기/페이지 이탈 시 API 호출 중단
- **다크모드 지원**
- **인쇄 전용 CSS** — `@media print` 최적화
- **반응형 레이아웃** — 모바일에서는 탭으로 전환

## 사용자 플로우

### 첫 사용자 온보딩 (5초 안에 이해 가능해야 함)
1. 랜딩 화면에서 "시작하기" 버튼 클릭
2. AI가 먼저 인사하며 "어떤 직무에 지원하세요?" 질문
3. 사용자가 직무 입력 → AI가 맞춤 질문 시작
4. 경력/경험을 대화로 입력
5. 실시간으로 우측 프리뷰가 채워짐

### 핵심 사용 플로우
```
1. 사용자: 직무/연차 입력
   ↓
2. AI: 경력 기반 질문 시작 (예: "가장 임팩트 있었던 프로젝트는?")
   ↓
3. 사용자: 자유롭게 경험 서술
   ↓
4. AI: STAR 기법으로 재구성 → Tool Use로 구조화된 JSON 출력
   ↓
5. 프리뷰 영역에 스트리밍으로 반영 (Diff 하이라이트)
   ↓
6. 사용자: JD 붙여넣기
   ↓
7. AI: 강조해야 할 섹션 제안 + 자동 재정렬 옵션 제공
   ↓
8. 사용자: PDF 내보내기
```

## 기술 스택

### 필수 스택
```
Framework:       Next.js 14 (App Router) + TypeScript
UI:              Tailwind CSS + shadcn/ui
상태관리:         Zustand
AI:              Anthropic Claude API (claude-sonnet-4)
                 - 스트리밍: SSE (Server-Sent Events)
                 - Tool Use: 구조화된 이력서 출력
PDF 변환:         @react-pdf/renderer
배포:            Vercel
```

### 선택 스택 (시간 여유에 따라 추가)
```
DB:              Supabase (이력서 저장/불러오기 기능 추가 시)
Auth:            Supabase Auth 또는 Clerk
애니메이션:       Framer Motion (Diff 하이라이트, 섹션 전환)
```

### Claude API 사용 이유
- 한국어 자연스러움이 우수하여 이력서 작성에 적합
- Tool Use 기능으로 구조화된 JSON 출력이 안정적
- 긴 컨텍스트 처리 능력이 좋아 이력서 전체 + JD 동시 분석에 유리

### 환경 변수
```env
ANTHROPIC_API_KEY=
NEXT_PUBLIC_APP_URL=
# 선택: Supabase 사용 시
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

5. **테마 선택** — 3~4가지 이력서 템플릿 중 선택

## 블로그 포스트 핵심 포인트 (면접 대비)

"기술적 결정들" 포스트에 반드시 포함할 5가지:

1. **왜 스트리밍을 선택했나**
   - 일반 요청 시 3초 대기 vs 스트리밍 시 즉시 반응 시작
   - 체감 대기시간 감소 효과 (본인 테스트 기준 수치 포함)

2. **Tool Use vs 프롬프트로 JSON 파싱**
   - Tool Use가 구조화된 출력에서 훨씬 안정적인 이유
   - JSON 파싱 실패 시 복구 전략

3. **상태 관리: 왜 Zustand인가**
   - Redux 대비 보일러플레이트 최소화
   - 이 규모의 앱에 적합한 이유

4. **Diff 하이라이트 구현**
   - 변경 전후 텍스트 비교 알고리즘 (`diff` 라이브러리 활용)
   - 스트리밍 중 실시간 계산 최적화

5. **프롬프트 엔지니어링**
   - STAR 기법을 어떻게 시스템 프롬프트로 녹였는가
   - 한국어 이력서 톤 조절 방법

## 작업 지침 (Claude Code 에이전트용)

### 개발 원칙
1. **이력서 JSON 스키마부터 설계할 것** — 모든 것의 기초. Zod로 타입 안전하게.
2. **Tool Use를 적극 활용할 것** — 프롬프트로 JSON 파싱 유도하지 말고 Tool Use 사용.
3. **스트리밍을 최우선으로 설계할 것** — 나중에 붙이면 구조 변경이 커짐.
4. **UX 디테일은 "일단 동작"보다 "어떻게 보이냐"가 중요** — 특히 로딩, 에러, 전환 상태.
5. **TypeScript strict mode 유지** — 타입 에러 무시 금지.
6. **환경 변수 관리** — API 키는 절대 클라이언트 번들에 포함되지 않도록 `/api` 라우트 경유.

### 작업 순서
각 Week를 순차적으로 진행하되, 매 Week 종료 시점에 다음을 수행:
- 해당 Week의 기능을 `README.md`에 스크린샷과 함께 문서화
- 주요 결정사항을 `DECISIONS.md`에 기록 (블로그 포스팅 재료)
- Git 커밋을 기능 단위로 나누어 작성

### 품질 기준
- 빌드 에러/경고 없을 것
- Lighthouse 점수 90+ (Performance, Accessibility, Best Practices)
- 주요 브라우저(Chrome, Safari, Firefox) 동작 확인
- 모바일 뷰포트에서 레이아웃 깨짐 없을 것

### 테스트 시나리오
최종 배포 전 아래 시나리오가 정상 동작해야 함:
1. 첫 방문 → 온보딩 → 이력서 1개 완성 → PDF 다운로드
2. JD 붙여넣기 → 강조 섹션 제안 확인 → 재정렬 적용
3. 스트리밍 중간에 뒤로가기 → API 호출 중단 확인
4. 섹션 드래그 → 순서 변경 → 저장 상태 유지
5. 다크모드 토글 → 모든 컴포넌트 정상 표시
